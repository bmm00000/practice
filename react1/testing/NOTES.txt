automated testing: code that tests our code (as opposed to manual testingp).

often, it's not so easy to differentiate between unit and integration tests, since very often you want to test a component that uses another component.

continue learning with the official documentation of jest and react testing library.

keep in mind that jest is not a tool focused on only react, it's as general javascript testing tool.

in the ecosystem section of react testing library, you can check extensions that you can install. some of these extensions are already part of the default set up in a react app (for example, the user-event extension), but there are other extensions of react testing library that is not mentioned in the docs, is the react hooks testing libirary, which makes testing hooks and especially custom hooks, way easier...
